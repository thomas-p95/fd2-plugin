## Related Guidelines

- `@clean` - Layer separation, dependency rules, package constraints
- `@state` - Cubit pattern, BLoC integration with DI
- `@dart` - Dart language best practices
- `@pubspec` - Always fetch latest versions from pub.dev before adding dependencies

## Overview

Projects use **get_it** as the service locator and **injectable** for annotation-driven code generation. All registrations are auto-generated by `build_runner` — no manual `sl.register*()` calls.

## New Project Setup

### 1. Add Dependencies

```yaml
# pubspec.yaml
# Check https://pub.dev/api/packages/<name> for latest version before adding
dependencies:
  get_it: <latest>
  injectable: <latest>

dev_dependencies:
  injectable_generator: <latest>
  build_runner: <latest>
```

### 2. Create Service Locator

```
lib/core/di/
├── di.dart                  # Barrel file
├── injection.dart           # Service locator setup
└── injection.config.dart    # Generated by build_runner
```

```dart
// lib/core/di/injection.dart
import 'package:get_it/get_it.dart';
import 'package:injectable/injectable.dart';
import 'injection.config.dart';

final sl = GetIt.instance;

@InjectableInit()
void configureDependencies() => sl.init();
```

```dart
// lib/core/di/di.dart (barrel) — only sl and configureDependencies are public API
export 'injection.dart' show sl, configureDependencies;
```

### 3. Package-Level DI (Micropackages)

Each package (`data`, `domain`) has its own injectable module using `@InjectableInit.microPackage()`. The main app auto-discovers and includes them.

#### Data Package

```yaml
# packages/data/pubspec.yaml
# Check https://pub.dev/api/packages/<name> for latest version before adding
dependencies:
  get_it: <latest>
  injectable: <latest>

dev_dependencies:
  injectable_generator: <latest>
  build_runner: <latest>
```

```
packages/data/lib/src/di/
├── di.dart                          # Barrel file
├── data_injection.dart              # Micropackage init
└── data_injection.config.dart       # Generated
```

```dart
// packages/data/lib/src/di/data_injection.dart
import 'package:injectable/injectable.dart';
import 'data_injection.config.dart';

@InjectableInit.microPackage()
initDataPackage() {} // generates DataPackageModule
```

```dart
// packages/data/lib/src/di/di.dart (barrel)
export 'data_injection.dart';
```

Export from package barrel:
```dart
// packages/data/lib/data.dart
export 'src/di/di.dart';
```

#### Domain Package

```
packages/domain/lib/src/di/
├── di.dart                            # Barrel file
├── domain_injection.dart              # Micropackage init
└── domain_injection.config.dart       # Generated
```

```dart
// packages/domain/lib/src/di/domain_injection.dart
import 'package:injectable/injectable.dart';
import 'domain_injection.config.dart';

@InjectableInit.microPackage()
initDomainPackage() {} // generates DomainPackageModule
```

```dart
// packages/domain/lib/src/di/di.dart (barrel)
export 'domain_injection.dart';
```

#### Main App Includes Packages

```dart
// lib/core/di/injection.dart
import 'package:get_it/get_it.dart';
import 'package:injectable/injectable.dart';
import 'injection.config.dart';

final sl = GetIt.instance;

@InjectableInit(
  externalPackageModulesBefore: [
    ExternalModule(DataPackageModule),
    ExternalModule(DomainPackageModule),
  ],
)
void configureDependencies() => sl.init();
```

`externalPackageModulesBefore` — data & domain register before app-level deps (use cases, cubits).
`externalPackageModulesAfter` — use when package deps depend on app-level registrations.

#### Generate All Packages

```bash
# Run build_runner in each package
cd packages/data && fvm dart run build_runner build --delete-conflicting-outputs
cd packages/domain && fvm dart run build_runner build --delete-conflicting-outputs

# Then main app
fvm dart run build_runner build --delete-conflicting-outputs
```

Or via melos:
```bash
fvm dart run melos run build_runner
```

### 4. Initialize in main

```dart
// lib/main.dart
void main() {
  configureDependencies();
  runApp(const App());
}
```

### 4. Generate Config

```bash
fvm dart run build_runner build --delete-conflicting-outputs
```

This generates `lib/core/di/injection.config.dart` with all registrations.

## Annotations

| Annotation | Scope | When to Use |
|------------|-------|-------------|
| `@injectable` | Factory (new instance per resolve) | Cubits, use cases |
| `@singleton` | Single instance, created immediately | App-wide services |
| `@lazySingleton` | Single instance, created on first use | Repos, heavy services |
| `@Injectable(as: Interface)` | Bind implementation to interface | Repos, use cases with abstract interface |
| `@module` | Group external/third-party bindings | SharedPreferences, Dio, etc. |
| `@factoryMethod` | Custom factory constructor | Named constructors |

## Registration Patterns

### API (Retrofit) — Data Layer

Retrofit APIs use factory constructors — can't annotate directly. Bind via `@module`:

```dart
// packages/data/lib/src/remote/api/feature/feature_api_module.dart
@module
abstract class FeatureApiModule {
  @lazySingleton
  FeatureApi featureApi(Dio dio) => FeatureApi(dio);
}
```

### Data Source — Data Layer

Annotate impl with `@Injectable(as: Interface)` — auto-discovered by micropackage init:

```dart
// Remote
@Injectable(as: FeatureRemoteDataSource)
final class FeatureRemoteDataSourceImpl implements FeatureRemoteDataSource {
  FeatureRemoteDataSourceImpl(this._api);
  final FeatureApi _api;
}

// Local
@Injectable(as: FeatureLocalDataSource)
final class FeatureLocalDataSourceImpl implements FeatureLocalDataSource {
  FeatureLocalDataSourceImpl(this._box);
  final Box _box;
}
```

Export interfaces only from barrel files:
- `lib/src/remote/remote.dart` — remote data source interfaces
- `lib/src/local/local.dart` — local data source interfaces

### Repository — Domain Layer

```dart
// packages/domain/lib/src/authentication/authentication_repository.dart
abstract interface class AuthenticationRepository {
  Future<void> login({required String email, required String password});
}

// packages/domain/lib/src/authentication/authentication_repository_impl.dart
@Injectable(as: AuthenticationRepository)
final class AuthenticationRepositoryImpl implements AuthenticationRepository {
  AuthenticationRepositoryImpl({
    required AuthenticationRemoteDataSource authenticationRemoteDataSource,
    required AuthenticationLocalDataSource authenticationLocalDataSource,
  })  : _authenticationRemoteDataSource = authenticationRemoteDataSource,
        _authenticationLocalDataSource = authenticationLocalDataSource;

  final AuthenticationRemoteDataSource _authenticationRemoteDataSource;
  final AuthenticationLocalDataSource _authenticationLocalDataSource;

  @override
  Future<void> login({required String email, required String password}) async {
    try {
      final tokens = await _authenticationRemoteDataSource.login(
        email: email,
        password: password,
      );
      await _authenticationLocalDataSource.save(
        accessToken: tokens.accessToken,
        refreshToken: tokens.refreshToken,
        tokenType: tokens.tokenType,
      );
    } on DataSourceException catch (e) {
      if (e.error?.name == ErrorName.unauthorized) {
        throw IncorrectCredentialException('Incorrect credential error');
      }
      throw LoginException(e.toString());
    }
  }
}
```

### Use Case — Domain Layer

```dart
// lib/use_case/account/determine_account/determine_account_uc.dart
abstract interface class DetermineAccountUseCase {
  Future<void> call();
}

// lib/use_case/account/determine_account/determine_account_uc_impl.dart
@Injectable(as: DetermineAccountUseCase)
final class DetermineAccountUseCaseImpl implements DetermineAccountUseCase {
  const DetermineAccountUseCaseImpl({
    required UserRepository userRepository,
    required SynchronizeInformationUseCase synchronizeInformationUseCase,
  })  : _userRepository = userRepository,
        _synchronizeInformationUseCase = synchronizeInformationUseCase;

  final UserRepository _userRepository;
  final SynchronizeInformationUseCase _synchronizeInformationUseCase;

  @override
  Future<void> call() async {
    final user = await _userRepository.getInformation();
    if (user.children.isNotEmpty) {
      await _userRepository.setMultipleUsersFlag(value: true);
      await _userRepository.removeCachedUser();
      return;
    }
    await _synchronizeInformationUseCase();
    await _userRepository.setMultipleUsersFlag(value: false);
  }
}
```

### Cubit — Presentation Layer

```dart
@injectable
class LoginCubit extends Cubit<LoginState> with CubitMixin<LoginState> {
  LoginCubit({
    required AuthenticationRepository authenticationRepository,
    required UserRepository userRepository,
    required DetermineAccountUseCase determineAccountUseCase,
  })  : _authenticationRepository = authenticationRepository,
        _userRepository = userRepository,
        _determineAccountUseCase = determineAccountUseCase,
        super(LoginState());

  final AuthenticationRepository _authenticationRepository;
  final UserRepository _userRepository;
  final DetermineAccountUseCase _determineAccountUseCase;
}
```

## Accessing Dependencies in Presentation Layer

Use `sl<T>()` inside `BlocProvider.create` to resolve dependencies. Never use `context.read<T>()` for repositories/use cases.

```dart
// CORRECT
BlocProvider(
  create: (_) => FeatureCubit(
    repository: sl<FeatureRepository>(),
    useCase: sl<DetermineAccountUseCase>(),
  )..load(),
  child: const FeatureScreen(),
)

// BAD — never resolve sl<T>() outside BlocProvider.create
Widget build(BuildContext context) {
  final repository = sl<FeatureRepository>(); // resolves every build
  ...
}

// BAD — never use context.read for repositories/use cases
final repository = context.read<Repository>();
```

## Dependency Rules

```
entity package      → NO dependencies
data package        → depends on entity only
domain package      → depends on entity, data (interfaces only)
lib/use_case        → depends on domain, entity
lib/screens         → depends on domain, use_case, entity

ENFORCE: No upward dependencies (domain never imports lib/screens)
```

## Testing

Override registrations in tests:

```dart
setUp(() {
  sl.reset();
  sl.registerFactory<FeatureRepository>(() => _MockFeatureRepository());
  configureDependencies();
});
```

## NEVER

- Register dependencies manually — use annotations + build_runner
- Use `context.read<T>()` for repositories/use cases — use `sl<T>()` in `BlocProvider.create`
- Inject data sources/APIs directly into cubits — go through repositories/use cases
- Create upward dependencies between packages
- Forget to run `build_runner` after adding/changing annotations

## Checklist: Adding New Dependency

### New Repository
- [ ] Create `abstract interface class` in domain package
- [ ] Create implementation with `@Injectable(as: Interface)`
- [ ] Translate `DataSourceException` to domain exceptions
- [ ] Export interface from domain barrel (hide impl)
- [ ] Run `build_runner` in domain package, then main app

### New Use Case
- [ ] Create `abstract interface class` with `call()` method
- [ ] Create implementation with `@Injectable(as: Interface)`
- [ ] Single responsibility — one business operation
- [ ] Inject only repositories and other use cases (no data sources/APIs)
- [ ] Export interface via feature barrel
- [ ] Run `build_runner`

### New Cubit
- [ ] Annotate with `@injectable`
- [ ] Inject only repositories and use cases via constructor
- [ ] Use `CubitMixin` and `safeEmit`
- [ ] Run `build_runner`

### New External Binding
- [ ] Create `@module` abstract class in appropriate package
- [ ] Use `@lazySingleton` or `@singleton` as appropriate
- [ ] Run `build_runner` in that package, then main app

### New Package (Micropackage)
- [ ] Add `get_it` + `injectable` to package `pubspec.yaml`
- [ ] Add `injectable_generator` + `build_runner` to dev_dependencies
- [ ] Create `lib/src/di/` with `@InjectableInit.microPackage()` init
- [ ] Create barrel `di.dart` exporting init function
- [ ] Export `di.dart` from package barrel
- [ ] Add `ExternalModule(PackageModule)` to main app `@InjectableInit`
- [ ] Run `build_runner` in package, then main app
